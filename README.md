## 4. 변경된 사항

 **화면 방향 변경 (세로 모드 -> 가로 모드)** <br/>
 세로 모드의 경우 공간이 부족해 플레이어가 분열하는 공을 피하기 어렵다고 판단하여 가로 모드로 게임을 개발하도록 목표를 변경하였습니다. <br/>
 또한 세로 모드의 경우 좌측 조이스틱과 우측 작살 발사 터치를 하기 위해서 화면을 너무 가려 시야가 답답해진다는 것도 목표 변경의 이유입니다. <br/> <br/>

 **스트라이프 애니메이션 사용 취소** <br/>
 스트라이프 애니메이션을 사용하기로 하였으나, 기존의 간단한 도형으로 만든 단순한 그래픽이 가시성이 더 좋고, 레트로한 느낌이 마음에 들어 변경하게 되었습니다.

## 5. 사용된 기술

**Lazy Initialization** <br/>
매 프레임마다 호출되는 Draw 내부에서 객체가 반복 생성되어 가비지 컬렉터로 인한 프레임 드랍이 발생 하지 않도록, 이를 방지하기 위해 Kotlin의 **by lazy**를 사용하여 최초 접근 시에만 한 번 메모리에 할당되도록 수업시간에 배웠던 Lazy Initialization를 적용했습니다. <br/> <br/>

**Collision Check**  <br/>
수업에서 다룬 AABB (RectF 교집합) 기반의 충돌 처리 로직을 적용하여 오브젝트들의 충돌 판정을 구현했습니다.

## 6. 아쉬운 것들 및 겪은 어려움

**하고 싶었지만 못한 것**:  <br/>Tiled MapEditor를 활용해 스테이지의 파괴 가능한 벽돌 지형을 배치해 보고 싶었으나 구현하지 못하였음. <br/>

**앱을 스토어에 판다면 보충할 것들**:  <br/>플레이 타임을 늘릴 수 있도록 스테이지를 좀 더 추가해야 한다. 또한 점수 랭킹 같은 것들도 도입해서 커뮤니티 기능도 활성화 시킬 수 있어야 할 것 같다. <br/>

**기말 프로젝트를 하면서 겪은 어려움**:  <br/>평소에 안드로이드 스마트폰을 사용하지 않고 아이폰을 사용했다 보니, 실제로 테스트할 안드로이드 기기가 없어 에뮬레이터로 테스트 하다 보니 실제 환경 테스트의 어려움이 있었다. <br/>
